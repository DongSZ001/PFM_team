# PF Assistant Directory Classification

Date: 2026-06-02
Project: PFM2 WebUI / PFM Team Assistant

## Purpose

This document classifies the current top-level contents of `pf_assistant/`. Use it before moving or deleting anything in that directory.

The main rule is: do not treat every top-level file as the same kind of thing. Some are runtime entry points, some are business modules, some are compatibility facades, and some are runtime state or installed dependencies.

## What Looks Like Historical Residue But Is Still Used

The root compatibility facades look small and old, but they are intentionally retained for older require paths and scripts:

- `gateway-config.js`
- `runtime-status.js`
- `unit-converter.js`
- `parameter-resolver.js`
- `parameter-definitions-seed.js`
- `material-parameters.js`

They should remain thin. Do not add implementation logic there.

## Cleanup Guidance

Safe next cleanup steps:

1. Keep adding classification before moving files.
2. Move only documentation first.
3. For runtime code, add tests before relocation.
4. Do not move `data/`, `logs/`, `start.env`, `node_modules/`, or `nanobot/` without deployment planning.
5. Keep compatibility facades until all callers are migrated and tested.

## Verification

For directory-structure changes, run:

```bash
node --check test/gateway-ui.test.js
node --test test/gateway-ui.test.js
```
